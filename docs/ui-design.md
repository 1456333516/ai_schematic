## 1. 设计系统

### 1.1 设计原则

| 原则 | 说明 | 落地方式 |
|------|------|---------|
| 专业工程感 | 界面风格对齐 EDA 工具用户认知 | 中性色为主、清晰的网格系统、精确的数值显示 |
| AI 自然融入 | AI 对话不抢占工作区、与画布操作无缝衔接 | 侧边栏对话面板、操作结果直接映射到画布 |
| 信息密度适中 | 避免 EDA 工具典型的信息过载问题 | 渐进式信息展示、上下文感知的面板显隐 |
| 操作可预期 | 用户操作后立即看到明确反馈 | 微动效确认、状态栏实时提示、undo/redo 可达 |

### 2.3 面板折叠行为

```
元器件库面板折叠
──────────────────────────────
展开态 → 点击折叠按钮 / Ctrl+Shift+L
  → 面板滑出（200ms ease-out）
  → 画布区域水平扩展占满
  → 左侧出现 16px 宽的唤出条（悬停显示展开图标）

AI 对话面板折叠
──────────────────────────────
展开态 → 点击折叠按钮 / Ctrl+Shift+R
  → 面板滑出（200ms ease-out）
  → 画布区域水平扩展占满
  → 右侧出现 16px 宽的唤出条（悬停显示展开图标 + AI 消息计数角标）

属性面板收起
──────────────────────────────
展开态 → 取消选中所有元器件 / 点击收起
  → 面板向下滑出（150ms ease-out）
  → 画布区域垂直扩展
```

**交互说明**：

| 操作 | 行为 |
|------|------|
| 搜索框输入 | 实时过滤元器件列表（模糊匹配名称/型号/关键词），300ms 防抖 |

**动态符号渲染（非内置元器件）**：

当 AI 生成或用户指定了不在内置库中的元器件时，系统按以下规则动态渲染：

| 步骤 | 行为 | 视觉反馈 |
|------|------|---------|
| AI 输出包含非内置元器件 | AI 提供 category/pins/properties → 动态渲染引擎接管 | 按 IC 通用矩形渲染（引脚数决定尺寸） |
| 渲染成功 | 以通用矩形符号 + 动态引脚布局显示 | 元器件边框内显示型号名称，引脚名标注在端口旁 |
| 渲染失败（引脚信息不完整） | 降级为占位符号 + 状态栏警告 | 黄色虚线边框矩形，内部显示 "?" + 型号名 |
| 用户双击占位符号 | 打开属性面板，手动补充引脚配置 | 属性面板出现"引脚编辑"区域 |

**元器件属性模板**：

不同类别的元器件具有不同的标准属性字段：

| 类别 | 必填属性 | 可选属性 | 校验规则 |
|------|---------|---------|---------|
| 电阻 | 阻值(value)、封装(footprint) | 功率(power)、精度(tolerance)、描述 | 阻值格式：数字+单位(Ω/KΩ/MΩ) |
| 电容 | 容值(value)、封装、耐压(voltage) | 介质类型、精度、描述 | 容值格式：数字+单位(pF/nF/μF) |
| 电感 | 感值(value)、封装 | 额定电流、DCR、描述 | 感值格式：数字+单位(nH/μH/mH) |
| 二极管/LED | 型号(model)、封装 | 正向压降、最大电流、描述 | — |
| 三极管/MOS | 型号(model)、封装 | 增益/阈值电压、最大电流、描述 | — |
| IC（通用） | 型号(model)、封装 | 供电电压、工作温度、描述 | 引脚数 ≥ 1 |
| 连接器 | 型号(model)、封装、引脚数 | 描述 | 引脚数 ≥ 1 |

**属性面板校验行为**：
- 必填属性为空时：输入框边框变红，下方提示"此字段为必填项"
- 格式错误时：输入框边框变红，下方提示具体格式要求
- 校验通过时：输入框恢复正常边框

#### 3.7.3 AI 响应状态

**流式数据契约**：

UI 层接收的 `DSLChunk` 需按以下规则映射到界面元素：

| DSLChunk.type | UI 映射 | 说明 |
|---------------|---------|------|
| `text` | 对话气泡内文字追加 | AI 的文字说明（如"正在分析..."） |
| `partial` | 进度条更新 + 元器件列表刷新 | 从 partial data 中提取已识别的 components 渲染为列表 |
| `complete` | 进度条 100% + 渲染画布 | 完整 DSL 交付 Domain Core |
| `error` | 显示错误气泡 | 错误信息展示 |

进度条百分比计算：由 Renderer 层根据已接收 `partial` 的 `data` 长度与预估总长度的比值推算（非服务端精确值），仅作视觉引导。

**流式生成中**：

```
┌─ AI ──────────────────────┐
│ 正在分析电路需求...          │   ← text chunk → 对话文字
│                            │
│ ████████████░░░░ ~75%      │   ← partial chunks 推算（近似值）
│                            │
│ 已识别元器件：               │   ← 从 partial.data.components 提取
│  • U1: AMS1117-3.3        │
│  • C1: 10μF 电解电容        │
│  • R1: 1KΩ 电阻            │
│ 正在计算连线布局...          │
│                            │
│           ┌──────┐         │
│           │ ⏹ 停止│         │   ← 绑定当前 requestId，调用 cancelStream
│           └──────┘         │
└────────────────────────────┘
```

**停止按钮与 requestId 绑定**：点击"停止"按钮时，UI 调用 `window.electronAPI.ai.cancelStream(requestId)` 取消当前请求。取消后对话气泡显示"已停止生成"，已接收的 partial DSL 不渲染到画布。

### 5.1 首次使用流程

```
启动应用
  │
  ├─ 初始化 Platform Layer（窗口、菜单）
  │    ※ 此阶段不初始化 AIService，避免无 API Key 时启动崩溃
  │
  ├─ 进入欢迎页
  │    │
  │    ├─ 检测 API Key 是否配置
  │    │    │
  │    │    ├─ 未配置 → 欢迎页顶部显示黄色提示条：
  │    │    │    "尚未配置 AI API Key，AI 功能不可用。[前往设置]"
  │    │    │    ※ 用户仍可进入项目进行手动编辑，AI 面板显示配置引导
  │    │    │
  │    │    └─ 已配置 → 延迟初始化 AIService（首次 AI 调用时按需创建）
  │    │
  │    ├─ 点击"新建项目" → 输入项目名称和路径 → 进入主编辑界面（空画布）
  │    │
  │    └─ 点击最近项目 → 加载项目数据 → 恢复画布 + 对话历史
  │
  └─ AI 面板首次交互（无 Key 时）
       │
       ├─ 用户在 AI 面板发送消息
       ├─ 检测 API Key → 未配置
       ├─ 弹出 API Key 设置对话框 → 配置并测试通过
       ├─ 按需初始化 AIService → 执行 AI 请求
       └─ 配置失败/取消 → AI 面板显示"请先配置 API Key"
```

**架构对齐说明**：AIService 采用延迟初始化模式，避免架构中 `new AIService()` 在无 Key 时直接抛错导致应用启动失败。应用启动仅初始化 Platform Layer 和 View Layer，AIService 在首次实际调用时才创建实例。

```
  │         │  │ │不保存 │ │ 保存并继续 │ │ 取消 │      │
  │         │  │ └──────┘ └──────────┘ └──────┘      │
  │         │  └──────────────────────────────────────┘
  │         │
  │         ├─ 点击"保存并继续" → 执行保存 → 继续原操作
  │         ├─ 点击"不保存" → 丢弃改动 → 继续原操作
  │         └─ 点击"取消" → 取消原操作，留在当前状态
```

**窗口标题栏状态指示**：当有未保存改动时，窗口标题显示 `● 项目名称 - AI Schematic Generator`（前缀圆点标识未保存状态）。

## 7. 动效规范

### 7.1 过渡动画

| 场景 | 动画 | 时长 | 缓动函数 |
|------|------|------|---------|
| 面板展开/折叠 | 水平滑动 | 200ms | `ease-out` |
| 属性面板展开/收起 | 垂直滑动 | 150ms | `ease-out` |
| 弹窗出现 | 缩放 + 淡入（从 95% → 100%） | 200ms | `ease-out` |
| 弹窗消失 | 缩放 + 淡出（100% → 95%） | 150ms | `ease-in` |
| Toast 出现 | 从上方滑入 + 淡入 | 300ms | `ease-out` |
| Toast 消失 | 淡出 | 200ms | `ease-in` |
| 画布缩放 | 平滑缩放 | 200ms | `ease-in-out` |
| 画布"适配视图" | 平滑平移 + 缩放 | 300ms | `ease-in-out` |

### 7.2 交互反馈动画

| 场景 | 动画 |
|------|------|
| AI 打字指示器 | 三个圆点依次上下跳动，循环播放 |
| AI 进度条 | 渐变蓝色条形推进，带微光扫过效果 |
| 元器件拖入画布 | 从拖拽位置 scale 0.8 → 1.0（100ms ease-out） |
| 元器件删除 | 淡出 + 轻微缩小（150ms） |
| AI 分析"定位"元器件 | 目标元器件选中框闪烁 2 次（蓝色 → 透明 → 蓝色，各 300ms） |
| 连线创建完成 | 连线从起点到终点快速描绘动画（200ms） |
| 保存成功 | 状态栏短暂显示 ✓ 图标（1.5s 后淡出） |
| 引脚吸附 | 引脚端口放大 1.5 倍 + 颜色加深（100ms） |

### 7.3 禁用动画场景

以下场景不使用动画，确保响应即时性：
- 元器件拖拽移动（实时跟随鼠标）
- 连线预览跟随（实时路由计算）
- 画布平移（实时跟随拖拽）
- 鼠标坐标更新
- 撤销/重做操作结果

## 9. 响应式与极端情况

### 9.1 窗口尺寸适配

| 窗口状态 | 布局策略 |
|---------|---------|
| 最小窗口（1024×600） | 元器件库面板自动折叠、AI 面板保留最小宽度 280px、属性面板覆盖画布 |
| 标准窗口（1440×900） | 三栏完整展示 |
| 最大化/大屏 | 画布区域扩展，面板宽度不变（除非用户手动拖拽调整） |

### 9.2 大型原理图性能

| 场景 | 策略 |
|------|------|
| 元器件 > 200 个 | 启用视口裁剪（仅渲染可见区域） |
| 元器件 > 500 个 | 缩小到 50% 以下时简化渲染（隐藏文字标注、简化符号） |
| 连线 > 500 条 | 分层渲染（远景只显示连线骨架） |

### 9.3 长对话处理

| 场景 | 策略 |
|------|------|
| 对话消息 > 50 条 | 虚拟滚动（仅渲染可见消息） |
| 单条消息过长 | 消息体最大高度 300px，展开按钮显示全文 |
| AI 分析报告过长 | 按严重等级分组折叠，默认展开"错误"组 |
