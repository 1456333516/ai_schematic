## 测试 1：元器件库渲染 (Tasks 15.9, 15.10)

**最先测这个，确认基础渲染没问题**

1. 打开应用，新建或打开一个项目，进入画布视图
2. 在左侧元器件库，展开每个分类，确认所有条目都能显示（不再是 "Coming soon"）
3. 在搜索框输入 `zener`，确认「稳压管」出现在结果中
4. 在搜索框输入 `npn`，确认「NPN三极管」出现
5. **拖拽测试**：把「电感」拖到画布 → 应显示 **三个弧形线圈** 符号
6. **拖拽测试**：把「NPN三极管」拖到画布 → 应显示 **BJT 符号**（竖线 + 两条斜臂）
7. **拖拽测试**：把「AND门」拖到画布 → 应显示 **D形体** 逻辑门
8. **双击测试**：双击「运算放大器 > 单运放」→ 应在画布中心添加 **三角形** OpAmp 符号
9. **双击测试**：双击「电源符号 > +3.3V」→ 画布添加后，符号标签应显示 `+3.3V`（不是 VCC）

**预期通过标准：** 所有元件可拖拽/双击添加，符号形状正确可识别

## 测试 3：电路分析功能 (Task 15.1)

1. 先用 AI 生成一个电路（例如输入「帮我生成一个 LED 驱动电路」）
2. 等 AI 生成完毕，画布有元件后
3. 点击顶部「**电路分析**」按钮
4. 按钮应变为 Loading 状态（旋转图标），等待 AI 返回
5. 完成后，聊天区域应出现一条**带颜色分区**的分析结果气泡：
   - 🔴 红色区域：错误（如有）
   - 🟡 黄色区域：警告（如有）
   - 🔵 蓝色区域：建议（如有）
   - 🟢 绿色「分析完成」：无问题时
6. 分析期间不能再次点击分析按钮（禁用状态）

**预期：** 结果气泡正常渲染，有颜色分区

## 测试 4：BOM 标签页显示 (Task 15.3)

1. 画布上有若干元件（电阻、电容、LED 等，最好有**多个同类型**元件）
2. 点击菜单 **Tools → Export → Export BOM...**，或按工具栏导出
3. 导出对话框打开，应有三个标签：**PNG | PDF | BOM**
4. 点击 **BOM** 标签
5. 确认显示一个表格，列为：序号 / 位号 / 名称 / 值 / 数量
6. 如果有两个阻值相同的电阻（如 R1, R2 都是 10k），它们应**合并为一行**，「位号」显示 `R1, R2`，「数量」显示 `2`

**预期：** BOM 表格正确聚合，同参数元件合并显示

## 测试 5：BOM 导出为 Excel (Task 15.4)

1. 接上一步，在 BOM 标签页中
2. 点击「**Export BOM (.xlsx)**」按钮
3. 弹出文件保存对话框，选择路径保存
4. 用 Excel 或 WPS 打开生成的 `.xlsx` 文件
5. 检查：
   - 工作表名称为 `BOM`
   - 第一行为**加粗表头**：序号 / 位号 / 名称 / 值 / 封装 / 数量 / 描述
   - 数据行与对话框中显示一致

**预期：** xlsx 文件结构正确，表头加粗

## 测试 6：版本快照自动创建 (Task 15.5)

1. 打开或新建一个已有路径的项目（必须先保存过）
2. 按 `Ctrl+S` 保存项目
3. 打开文件资源管理器，导航到**项目文件所在目录**
4. 应能看到一个 `.aischematic_versions` 文件夹
5. 进入该文件夹，应有一个以时间戳命名的 `.json` 文件（如 `1741234567890.json`）
6. 随便打开这个 json 文件，内容应是完整的项目数据（含 `schema`、`canvas`、`meta`）

**预期：** 每次保存在版本目录自动生成快照文件

2. 点击菜单 **File → Version History**

   - 文件大小（KB）
   - 「恢复」按钮

**预期：** 抽屉正确显示，时间线按最新在上排列

## 测试 9：版本恢复 (Task 15.8)

1. 接上一步，在 Drawer 中选择**一个旧版本**（列表中靠下的）
2. 点击该条目的「**恢复**」按钮
3. 弹出确认对话框「确认恢复到该版本？」
4. 点击「确定」
5. 观察画布和元件是否切换到该版本的状态
6. 应弹出 `已恢复到该版本` 的成功提示
7. Drawer 自动关闭
8. 标题栏应显示 `●` 脏标记（因为恢复后未保存）

**预期：** 画布内容回到所选版本，状态正确
