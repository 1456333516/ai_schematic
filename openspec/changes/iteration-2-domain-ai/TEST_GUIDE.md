# Iteration 2 功能测试指南

## 测试环境准备

### 前置条件
1. 已安装 Node.js (v16+)
2. 已安装项目依赖：`npm install`
3. 拥有有效的 Claude API Key（格式：`sk-ant-...`）

### 启动应用
```bash
npm run dev
```

## 附录：常见问题

### Q1：为什么 AI 生成的组件不显示在画布上？
**A**：Canvas Integration 未完全实现。虽然 Domain Core 和 GraphSyncer 已经实现，但事件处理器的重构未完成，导致 AI 生成的组件无法正确同步到 X6 画布。

### Q2：为什么无法测试 Undo/Redo 功能？
**A**：Undo/Redo 功能依赖于 Command Bus，而 Command Bus 需要通过事件处理器触发。由于 Canvas Integration 未完全实现，手动操作不会触发命令，因此无法测试 Undo/Redo。

### Q3：如何验证 Domain Core 是否正常工作？
**A**：可以通过浏览器开发者工具的 Console 查看日志，确认没有错误。也可以通过 React DevTools 查看组件树，确认 DomainProvider 正确初始化。

### Q4：测试时需要真实的 API Key 吗？
**A**：大部分 UI 测试不需要真实的 API Key。但测试 AI 生成功能时，需要有效的 Claude API Key 和网络连接。

### Q5：如何获取 Claude API Key？
**A**：访问 https://console.anthropic.com/ 注册账号并创建 API Key。
