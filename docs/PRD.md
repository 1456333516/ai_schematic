## 1. 产品概述

### 1.1 产品愿景

通过 AI 大模型驱动，将自然语言描述转化为专业级电路原理图，大幅降低电路设计门槛，提升设计效率。

### 1.2 产品定位

一款 **Windows 桌面端** 的 AI 电路原理图生成与编辑工具。用户通过自然语言描述电路需求，AI 自动生成符合嘉立创 EDA 符号标准的原理图，并提供完整的画布编辑、AI 分析、项目管理与多格式导出能力。

**边界定义**：

- **包含**：原理图（Schematic）的 AI 生成、编辑、分析、导出
- **不包含**：PCB Layout 设计、电路仿真、Gerber 文件生成

### 1.3 目标用户

| 用户群体 | 核心诉求 | 使用场景 |
|---------|---------|---------|
| 电子工程师 | 快速生成常规电路模块，提升设计效率 | 项目前期快速原型验证 |
| 高校学生 | 降低电路设计学习门槛 | 课程设计、毕业设计 |
| 硬件爱好者/Maker | 零基础也能生成可用的原理图 | DIY 项目、创客活动 |
| 技术管理者 | 快速理解和审查电路方案 | 方案评审、技术沟通 |

## 2. 功能需求

### 2.1 F1 - AI 原理图生成

**优先级**：P0（核心功能）

**描述**：用户通过自然语言描述电路需求，AI 自动生成完整的电路原理图。

**功能规格**：

| 编号 | 需求项 | 详细说明 |
|------|-------|---------|
| F1.1 | 自然语言输入 | 用户在 AI 对话面板输入电路描述，支持中英文 |
| F1.2 | AI 生成原理图 | AI 解析用户意图，输出结构化电路描述（中间 DSL/网表），经校验后渲染到画布 |
| F1.3 | 生成过程可视化 | 生成过程中显示进度状态，支持流式输出反馈 |
| F1.4 | 嘉立创 EDA 符号标准 | 生成的原理图元器件符号严格遵循嘉立创 EDA 标准 |
| F1.5 | 自动布局 | AI 生成的原理图自动进行合理的元器件布局和连线排布 |

**AI 输出流水线**：

```
用户自然语言 → AI 模型 → 中间 DSL/网表 → 校验管线 → Domain Core → X6 渲染
```

**典型用例**：

```
用户输入："帮我设计一个基于 AMS1117-3.3 的 5V 转 3.3V LDO 稳压电路，
         包含输入输出滤波电容和指示 LED"

AI 生成：包含 AMS1117-3.3、输入 10μF 电解电容、输出 10μF 电解电容、
        100nF 陶瓷去耦电容、LED 指示灯、限流电阻的完整原理图
```

### 2.2 F2 - 画布编辑器

**优先级**：P0（核心功能）

**描述**：提供完整的原理图画布编辑能力，用户可在 AI 生成的基础上进行手动修改。

**功能规格**：

| 编号 | 需求项 | 详细说明 |
|------|-------|---------|
| F2.1 | 画布基础操作 | 缩放（滚轮）、平移（拖拽空白区域）、全局视图适配 |
| F2.2 | 元器件拖拽 | 选中并拖拽移动元器件，支持多选批量移动 |
| F2.3 | 元器件添加/删除 | 从元器件库拖入新元器件到画布；选中后 Delete 键删除 |
| F2.4 | 连线编辑 | 从引脚拖出创建连线；拖拽连线折点修改路径；删除连线 |
| F2.5 | 参数编辑 | 双击元器件打开属性面板，修改阻值/容值/型号等参数 |
| F2.6 | 网络标签 | 添加/编辑/删除网络标签（Net Label），用于跨区域电气连接 |
| F2.7 | 电源符号 | 添加 VCC、GND、+5V、+3.3V 等电源/地符号 |
| F2.8 | 多页原理图 | 支持多页 Sheet 管理，页间导航切换，跨页网络连接 |
| F2.9 | Undo/Redo | 基于命令模式的撤销/重做，支持完整的操作历史 |
| F2.10 | 对齐与网格 | 元器件吸附网格，支持对齐辅助线 |
| F2.11 | 框选与批量操作 | 矩形框选多个元器件，批量移动/删除/复制 |
| F2.12 | 复制/粘贴 | 选中元器件和连线后复制粘贴 |

### 2.3 F3 - AI 对话式交互

**优先级**：P0（核心功能）

**描述**：侧边栏常驻 AI 对话面板，用户边查看原理图边与 AI 对话，支持迭代修改。

**功能规格**：

| 编号 | 需求项 | 详细说明 |
|------|-------|---------|
| F3.1 | 侧边栏对话面板 | 右侧常驻 AI 对话面板，可折叠/展开，不遮挡画布 |

### 2.4 F4 - AI 原理图分析

**优先级**：P1

**描述**：AI 对当前原理图进行多维度分析，提供专业建议。

**功能规格**：

| 编号 | 需求项 | 详细说明 |
|------|-------|---------|
| F4.1 | 电路功能正确性检查 | 分析电路拓扑是否符合设计意图，检测逻辑错误 |
| F4.2 | 元器件参数合理性 | 检查阻值/容值/耐压等参数是否在合理范围（如去耦电容容值是否足够） |
| F4.3 | 连线错误检测（DRC） | 检测悬空引脚、短路、未连接的网络标签等电气规则违规 |
| F4.4 | 电路优化建议 | 提供改进建议（如增加保护电路、优化滤波参数等） |
| F4.5 | 分析报告 | 以结构化形式展示分析结果，按严重程度分级（错误/警告/建议） |

## 4. 技术架构

### 4.1 技术栈

| 层次 | 技术选型 | 选型依据 |
|------|---------|---------|
| 桌面框架 | Electron | EDA 领域验证（嘉立创 EDA 采用）；社区最大，问题排查资源最丰富 |
| 前端框架 | React 18+ / TypeScript | 组件化、强类型、AntV X6 官方 React 支持 |
| 图编辑引擎 | AntV X6 | 内置节点/边/端口/路由原语，天然适配原理图场景；SVG 渲染利于矢量导出 |
| 状态管理 | Zustand | 轻量、支持 undo/redo 中间件、低耦合 |
| AI 集成 | Anthropic SDK (TypeScript) | Claude 流式输出；Adapter 模式预留多模型扩展 |
| 导出 PNG/PDF | 领域模型 → SVG → 光栅化/矢量化 | 基于领域模型而非 DOM 截图，保证输出质量 |
| 导出 BOM | ExcelJS | 成熟的 Excel 生成库 |
| 数据格式 | JSON | 人类可读、利于版本 diff |
| 版本管理 | 快照 + 操作增量 + 语义化 diff | 工程级版本控制 |
| UI 组件库 | Ant Design 5.x | 与 X6 同生态、企业级组件库 |

### 4.3 核心架构原则

| 原则 | 实现方式 |
|------|---------|
| **关注点分离** | Main Process（I/O、安全）与 Renderer Process（UI、交互）通过 IPC 解耦 |
| **领域驱动** | 电路"真相"存在 Domain Core 中，X6 只是视图层投影 |
| **命令模式** | 所有画布操作通过 Command Bus 调度，天然支持 undo/redo 和操作日志 |
| **适配器模式** | AI 引擎接口标准化，新增模型只需实现 `AIProvider` 接口 |
| **策略模式** | 导出功能标准化，新增格式只需实现 `ExportStrategy` 接口 |
| **数据驱动** | 元器件符号定义（JSON）与渲染逻辑分离，可独立更新 |
| **渲染器可替换** | X6（SVG）为当前实现，架构预留切换到 Canvas/WebGL 渲染引擎的能力 |

### 4.4 AI 集成架构

**AI Provider 接口定义**：

```typescript
interface AIProvider {
  id: string;
  name: string;

  // 生成原理图
  generateSchematic(prompt: string, context?: SchematicContext): AsyncIterable<DSLChunk>;

  // 修改原理图
  modifySchematic(instruction: string, currentState: SchematicDSL): AsyncIterable<DSLChunk>;

  // 分析原理图
  analyzeSchematic(schematic: SchematicDSL): Promise<AnalysisReport>;
}
```

**AI 输出约束**：

- AI 不直接操作画布，而是输出受约束的中间 DSL/网表
- DSL 经 Validation Pipeline 校验和修复后，再同步到 Domain Core
- Domain Core 状态变更后，X6 视图层自动更新

**中间 DSL 示例**：

```json
{
  "components": [
    {
      "id": "U1",
      "type": "AMS1117-3.3",
      "category": "LDO",
      "properties": { "input": "5V", "output": "3.3V" },
      "pins": ["VIN", "VOUT", "GND"]
    },
    {
      "id": "C1",
      "type": "capacitor_electrolytic",
      "properties": { "value": "10uF", "voltage": "16V" }
    }
  ],
  "connections": [
    { "from": "U1.VIN", "to": "C1.1", "net": "VIN_5V" },
    { "from": "U1.GND", "to": "C1.2", "net": "GND" }
  ],
  "netLabels": [
    { "name": "VIN_5V", "type": "power" },
    { "name": "GND", "type": "ground" }
  ]
}
```


**原理图数据模型核心字段**：

```typescript
interface SchematicSheet {
  id: string;
  name: string;
  components: Component[];      // 元器件列表
  wires: Wire[];                // 连线列表
  netLabels: NetLabel[];        // 网络标签
  powerSymbols: PowerSymbol[];  // 电源符号
  metadata: SheetMetadata;      // 页面元信息
}

interface Component {
  id: string;                   // 位号（R1, C1, U1）
  libraryId: string;            // 元器件库 ID
  type: string;                 // 元器件类型
  position: { x: number; y: number };
  rotation: number;             // 旋转角度（0/90/180/270）
  properties: Record<string, string>; // 参数（值/封装/描述等）
  pins: Pin[];                  // 引脚列表
}

interface Wire {
  id: string;
  points: { x: number; y: number }[]; // 折线路径点
  net: string;                  // 所属网络名
  startPin?: PinRef;            // 起始引脚引用
  endPin?: PinRef;              // 终止引脚引用
}
```

### 5.2 核心交互流程

**流程 1：AI 生成原理图**

```
启动应用 → 创建新项目 → 在 AI 面板描述电路需求
→ AI 流式生成中间 DSL → 校验通过 → 渲染到画布
→ 用户查看/编辑 → 继续对话迭代修改
```

**流程 2：手动编辑**

```
从元器件库拖入元器件 → 放置到画布 → 连接引脚
→ 编辑参数 → 添加网络标签/电源符号
→ AI 分析检查 → 修正问题 → 导出
```

**流程 3：版本管理**

```
编辑操作 → 自动记录操作增量 → 定期生成快照
→ 查看版本历史 → 选择版本对比/回退
```

## 7. 风险与应对

| 风险 | 可能性 | 影响 | 应对策略 |
|------|-------|------|---------|
| AI 生成的原理图质量不稳定 | 高 | 高 | Validation Pipeline 校验 + 修复循环；用户可手动修正 |
| SVG 渲染大型原理图性能瓶颈 | 中 | 中 | 视口裁剪 + 增量渲染；架构预留渲染引擎切换 |
| AntV X6 缺少 EDA 专用功能 | 高 | 中 | 自建 Domain Core 补充 EDA 语义；X6 仅作视图层 |
| Claude API 调用成本和速率限制 | 中 | 中 | 本地缓存 + 请求合并；提供 API Key 自配能力 |
| 嘉立创 EDA 符号标准变更 | 低 | 低 | 符号定义数据驱动，可独立更新 |
